## Projekti ülevaade

Projekt demonstreerib käsitsi API testimist, kasutades riskipõhist testimisviisi.

Eesmärgiks oli testida REST ja SOAP API-sid, hinnata süsteemi käitumist erinevate sisendite korral ning tuvastada funktsionaalseid vigu ja süsteemi piiranguid.

## Testimise fookus

Funktsionaalsuse testimine
Positiivsed ja negatiivsed stsenaariumid
Veahalduse
API vastuste ja HTTP staatusekoodide valideerimine
Riskide arvestamine testjuhtumite loomisel

## Testimise eesmärgid

Kontrollida API vastavust dokumentatsioonile
Analüüsida süsteemi käitumist korrektsete ja vigaste sisenditega
Verifitseerida HTTP vastuskoodid ja API vastused
Tuvastada loogikavead ja süsteemi piirangud
Harjutada reaalse QA tööprotsessi rakendamist
